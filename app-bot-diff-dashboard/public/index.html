Claude App Bot's Diff Pipeline

Every commit to the hand-built app is rebuilt by the Claude bot, diffed against the original, and surfaced back as a PR so the bot keeps learning.

weresobach → bot → bottest → audit → bridge
How the pipeline works
●
Commit
A new commit lands on weresobach/main (the hand-built north-star app).
●
Phase 2 sync
path-b-sync workflow sends the commit + diff to the bridge's /prompt API so the bot rebuilds the same feature on bottest.
●
Bottest commit
The bot pushes its output to weresobachbottest/main with a [path-b-sync] tag.
●
Phase 3 audit
gap-audit workflow fingerprints bottest against 7 curated patterns and reports missing features.
●
Gap PR
If gaps exist, a draft PR lands on discord-claude-app-builder with the findings to improve bot prompts.
